Per il codice C abbiamo deciso di adottare le seguenti scelte progettuali:
Per facilitare il debugging, migliorare l'ordine e la pulizia del codice abbiamo suddiviso il problema principale in più sottoproblemi utilizzando funzioni separate.
Sempre per migliorare la pulizia del codice abbiamo utilizzato file .h per esplicitare solo la firma delle funzioni con relative descrizioni del loro funziomento e file .c per la loro effettiva implementazione.
Il file main.c si occupa esclusivamente di recuperare gli argomenti passati da terminale, inizializzare le variabili globali, inizializzare le strutture (frames e liste) e gestire i thread.
All'interno di utility.c invece si trovano tutte le funzioni necessarie per l'applicazione degli algoritmi e per la gestione dei file

Per risolvere alcune problematiche abbiamo implementato le seguenti funzionalità:
-  Indichiamo la RAM vuota con page_id = -1 per sapere se ci sono frames occupabili
-  Le funzioni degli algoritmi ritornano se c'è stato o meno un page fault per aumentare un contatore
-  Se una riga letta dal file di input non è valida viene saltata dal programma

Per lo script bash invece:
-  Controlliamo l'esistenza dei file essenziali per l'esecuzione del programma (main.c e utility.c)
- "Catturiamo" l'output del programma con le statistiche scritte dal file .c su un file creato all'avvio dell'esecuzione ed eliminato alla sua conclusione
